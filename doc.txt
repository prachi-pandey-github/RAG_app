==================================================
1) PROJECT OVERVIEW
==================================================
This project performs:
- PDF to image conversion
- OCR text extraction (Tesseract by default, optional TrOCR)
- Document summarization using Groq-hosted Llama model
- Optional object detection/cropping of machine-related objects from PDF pages using Grounding DINO

Core app files:
- main.py           -> FastAPI app entrypoint
- api.py            -> FastAPI API (same structure as main.py)
- trocr.py          -> OCR + summarization pipeline
- yolo.py           -> PDF page object detection and crop extraction
- requirements.txt  -> declared Python dependencies
- Dockerfile        -> container setup

5) MODELS USED + VERSION/CHECKPOINT DETAILS
==================================================
A) Object Detection (Grounding DINO)
- Model type: Zero-shot object detection
- Framework: Hugging Face Transformers
- Model ID (checkpoint): IDEA-Research/grounding-dino-base
- Loaded in: yolo.py
- Notes: Uses text prompt to detect machine/equipment/electronics objects in PDF page images.

B) OCR Model (Optional TrOCR path)
- Model type: Transformer OCR (VisionEncoderDecoder)
- Framework: Hugging Face Transformers
- Model ID (checkpoint): microsoft/trocr-base-printed
- Loaded in: trocr.py when use_tesseract=False
- Notes: Suitable for line/image OCR; currently optional in code.

- Loaded in: trocr.py when use_tesseract=True (default)

- Provider SDK: groq
- Model name used: llama-3.3-70b-versatile
- Called in: trocr.py function summarize_with_llama()
- Notes: Requires GROQ_API_KEY in .env or environment variable.

- Notes: Legacy/commented YOLO code exists in yolo.py; active implementation currently uses Grounding DINO.

==================================================
7) API / APPLICATION SPEC
==================================================
FastAPI application:
- App title: TrOCR PDF API
- App version: 1.0.0
- Endpoint: POST /ocr

/ocr input parameters:
- file: UploadFile (PDF only)
- use_tesseract: bool (default True)
- dpi: int (default 300)
- include_text: bool (default False)

/ocr behavior:
1. Validates uploaded file is PDF
2. Converts PDF pages to images
3. Runs OCR per page
4. Summarizes extracted text using Groq Llama model
5. Returns summary (and optionally full text)
